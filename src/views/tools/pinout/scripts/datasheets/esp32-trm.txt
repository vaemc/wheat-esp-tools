Notes:
  * The original 6.10-1 table has a "Pad Name" column between "GPIO#"
    and "Function0". It has been dropped here so that rows match the
    parser shape used by every other chip ("GPIO# F0 F1 F2 F3 F4 F5
    DRV RST Note"). RST is left as "-" because the ESP32 TRM only
    publishes a single combined "Reset" column; the reference info is
    preserved in the trailing Note column.
  * Table 6.11-1 is split into two virtual tables (RTC IO MUX vs.
    Analog) so that RTC routing and analog channels land in the
    correct UI sections.
  * GPIO20, 24, 28-31 are intentionally absent — they don't exist on
    the ESP32 chip.

DRV (default IO drive strength):
  0:~5mA  1:~10mA  2:~20mA  3:~40mA
RST (reset state) — see Table 6.10-1 footnote in the ESP32 TRM:
  R: routed through RTC IO MUX
  I: GPIO34-39 are input-only

The sar_i2c_* functions are the ULP RTC I2C on GPIO0/2/4/15. ESP32
has a single ULP I2C; any of these pads can be assigned via the
RTCIO_SAR_I2C_IO_REG register.
